This is a test of Sass

Sass is a meta-language on top of CSS that's used to describe the style of a document cleanly and structurally, with more power than flat CSS allows. Sass both provides a simpler, more elegant syntax for CSS and implements various features that are useful for creating manageable stylesheets.
Click to learn more about Sass

    Reasons to use Sass

  • Nesting keeps things organized.
  • Parent reference is cool!
  • Variables promote DRY.
  • Operations are interesting.
  • Mix-ins make it all worth it.
  • Arguments promote DRY.
  • Imports keep things organized.
  • Generated CSS will be syntactically correct.

    Reasons not to use Sass

  • It's yet another syntax.
  • One more layer of complexity.
  • Many designers don't think in code.
